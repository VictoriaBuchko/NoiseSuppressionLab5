## Реалізація
 
У класах фільтрів реалізовано методи update, які імітують роботу в real-time режимі - кожен новий вимір обробляється окремо, без доступу до майбутніх точок
 
### SMA (Simple Moving Average)
Лінійний фільтр із ковзним вікном. Реалізовано з оптимізацією O(1) через підтримку поточної суми:
 
```python
def update(self, x):
    if len(self.q) == self.w:
        self.sum -= self.q[0]
    self.q.append(x)
    self.sum += x
    return self.sum / len(self.q)
```
 
### EMA (Exponential Moving Average)
Рекурсивний фільтр, який зберігає лише одне попереднє значення:
 
```python
def update(self, x):
    if self.last is None:
        self.last = x
    else:
        self.last = self.a * x + (1 - self.a) * self.last
    return self.last
```
 
### Median Filter
Нелінійний фільтр - повертає медіану поточного вікна спостережень:
 
```python
def update(self, x):
    self.q.append(x)
    return np.median(self.q)
```

### Аналіз результатів
 
**Чи сильно лінії фільтрів відстають від чорного пунктиру (Ідеалу)?**

На графіку 3 в ділянці 13–20 секунд усі три фільтри помітно відстають від чорного пунктиру. SMA і медіана мають вікно ~20–21 точка, що дає затримку близько 0.4 секунди. EMA з α=0.1 поводиться схоже — еквівалентне вікно теж близько 20 точок. На піках синусоїди всі лінії трохи "зрізають кути" і не встигають за ідеалом, але загалом відставання помірне і прийнятне

**Як медіанний фільтр впорався з поодинокими "викидами"? Порівняйте з SMA.**

Медіанний фільтр практично не реагує на поодинокі різкі викиди — зелена лінія залишається рівною навіть там, де сірі точки різко стрибають вгору чи вниз. SMA натомість включає кожен викид в усереднення і "розмазує" його ефект на наступні 20 точок, через що на синій лінії після кожного стрибка помітне хвилеподібне спотворення

### Аналіз спектру помилки (Графік 4b)

**Що сталося з кольоровими лініями помилки у зоні низьких частот (0–1 Гц)? Чи стали вони вищими за сіру лінію?**

На графіку 4b у зоні 0–1 Гц кольорові лінії SMA та EMA піднімаються вище за сіру лінію вхідного шуму. Тобто помилка фільтрованого сигналу на низьких частотах стала більшою, ніж помилка нефільтрованого — це і є парадокс надмірного згладжування

**Поясніть феномен: чому придушення шуму призвело до збільшення помилки на низьких частотах?**

Траєкторія "змійки" має частоту 0.5 Гц, а SMA з вікном 100 точок пам'ятає рівно 2 секунди (100/50 = 2 с) і запізнюється на 1 секунду — більше за половину одного циклу. Фільтр показує не реальне положення, а усереднення за минулі 2 секунди, що породжує систематичну помилку в кілька метрів при шумі лише 0.8 м. На низьких частотах фільтр не прибирає похибку, а сам її створює через запізнення — це і відображається як "горб" зліва на спектрі

### Аналіз

**Чи зникли "широкі" викиди (якщо шум триває кілька точок підряд)?**

Поодинокі викиди медіана з вікном 5 все ще пригнічує добре. Але якщо аномалія триває 3 і більше точок підряд — вона може пройти крізь фільтр, бо медіана з п'яти значень вибирає середнє, і якщо більшість точок у вікні вже спотворені, результат теж буде спотворений. "Широкі" викиди фільтр повністю не усуває

**Порівняйте гладкість лінії Median з лінією SMA при аналогічному вікні**

За гладкістю медіана W=5 поступається SMA з аналогічним вікном — зелена лінія виглядає більш "ламаною" і стрибкоподібною, бо медіана щоразу вибирає конкретне значення з вікна, а не плавне середнє. SMA при W=5 дає рівнішу криву, але гірше справляється з викидами

## Висновок
 
(а) Видалення збоїв сенсора:
Найкраще підходить медіанний фільтр, тому що він не реагує на різкі одиничні стрибки. Він бере середнє по порядку, тому один великий викид не впливає на результат. А от SMA та EMA враховують всі значення, тому викид розтягується і ще деякий час псує сигнал

(б) Плавне ведення траєкторії:
Тут краще працюють SMA або EMA з нормальним вікном. Вони роблять рух більш рівним і без різких коливань. EMA зазвичай трохи краще, бо реагує швидше. Але якщо зробити занадто велике вікно, то траєкторія стає запізненою і може спотворювати реальний рух
